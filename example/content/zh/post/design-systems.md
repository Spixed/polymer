一个精心设计的设计系统是任何可扩展产品的基石。让我们探索它们的运作方式。

## 核心原则

设计系统不仅仅是组件库——它们是设计决策的**活文档**。

### Token 架构

在基础层面，我们有设计 tokens：

```css
:root {
    --color-primary: #2979FF;
    --spacing-unit: 8px;
    --radius-medium: calc(var(--spacing-unit) * 2);
}
```

这些 tokens 驱动着从按钮到整个布局的一切。

## 组件层次结构

1. **原子**：基础构建块（按钮、输入框）
2. **分子**：原子的组合（搜索栏）
3. **有机体**：复杂的 UI 部分（导航）
4. **模板**：页面级结构
5. **页面**：具体实现

> 目标不是完美——而是**一致性**。

好的设计通常遵循数学比例：

$$
\text{比例系数} = \frac{f_{n+1}}{f_n} \approx 1.618
$$
